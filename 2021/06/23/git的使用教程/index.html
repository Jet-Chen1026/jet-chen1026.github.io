git分支管理

可以把一条分支看做一条时间线，主分支名为master，所有的修改最终都应该提交到master上，但是为了避免失误，应该有一条辅助分支，比如说tmp分支，用于暂时保存团队的开发成果，当确定没有问题的时候再将tmp分支上的内容提交给master分支，同时每一个团队开发者应该有一条属于自己的分支，比如teamworker1，所有团队成员在自己的分支上修改项目，每次将修改之后将修改提交给tmp分支，当最后整个团队认为tmp分支的内容可以成为下一个release版本的时候，再将tmp分支的内容提交给master分支。这就是团队使用git的流程

下载远程代码库之后应该首先做这几件事：

  • 查看所有分支，以及目前正处于哪一条分支：git branch
  • 如果没有属于你自己的开发分支，则新建一条属于自己的分支：
```
git branch dev   创建分支

git checkout dev   将开发的时间线转移到dev分支上
```
  • 开始修改你的代码

提交与更新

本地

当你在本地修改了代码之后首先使用git add filename，将修改提交给缓存区，此时修改并没有完全提交给本地仓库，如果需要完全提交给本地仓库，还需要使用git commit -m “本次修改的说明”来将你提交给缓存区的修改提交给本地仓库

当别人修改了代码，并提交到远程仓库的时候，你的本地仓库就与远程仓库的内容不一样了，这个时候要先在本地git库内执行：
git pull
将远程修改内容同步到本地

远程

当你本地修改代码之后，并且已经使用git add与git commit上传到你本地版本库之后，使用：
git push origin 分支名
按理来说只需要先提交你自己的分支，如果你的分支叫dev，则：
git push origin dev
